# Ресурсы под faster-whisper

Оценки для CPU-инференса с `compute_type="int8"` (наш дефолт). RTF — real-time factor: доля секунды процессорного времени на секунду аудио; для реального времени нужен RTF заметно < 1.

| Модель | Диск (int8) | RAM | RTF на 8-ядерном desktop CPU | Качество |
|--------|------------|-----|------------------------------|----------|
| tiny   | ~75 МБ  | ~0.5 ГБ | ~0.05–0.1 | слабое, только английский приемлем |
| base   | ~140 МБ | ~0.7 ГБ | ~0.1–0.15 | минимум для русского |
| small  | ~460 МБ | ~1.5 ГБ | ~0.2–0.4  | **дефолт проекта** — рабочий баланс |
| medium | ~1.5 ГБ | ~3.5 ГБ | ~0.6–1.0  | на грани реального времени на CPU |
| large-v3 | ~3 ГБ | ~5.5 ГБ | >1 | только с GPU |

Дополнительно:

- **Два потока транскрипции не нужны:** loopback и микрофон буферизуются раздельно, но инференс у нас последовательный — одна модель, пики CPU кратковременные (реплика ~3 с аудио → ~1 с инференса на `small`).
- **GPU (CUDA):** `device="cuda", compute_type="float16"` даёт RTF ~0.02–0.05 даже на large; medium/large становятся доступны. VRAM: small ~1 ГБ, medium ~2.5 ГБ, large ~4.5 ГБ.
- **Первый запуск** скачивает модель с HuggingFace в `%LOCALAPPDATA%` — нужен интернет и место по таблице.
- **Загрузка модели** (one-time): 1–5 с для small на CPU. Держим процесс живым всю сессию, поэтому на латентность ответа не влияет.
- **Итоговый минимум для MVP** (small, int8, CPU): ~2 ГБ свободной RAM, 4+ ядра, ~500 МБ диска. Рекомендуется: 8 ядер / 16 ГБ RAM — тогда транскрипция не конкурирует с клиентом видеовстречи.

Замер на своей машине: `/latency-check` (см. `.claude/skills/latency-check`).
