# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Проект: помощник для собеседований

Цель: считывать звук встречи (системный аудиовыход) и микрофона, транскрибировать его и отвечать на вопросы **в реальном времени**.

## Команды

```powershell
uv sync                                            # установка зависимостей
uv run pytest -q                                   # все тесты
uv run pytest -q tests/test_answer.py -k question  # один тест
uv run ruff format . ; uv run ruff check --fix .   # формат + линт
uv run python -m interview_helper.web              # веб-UI на http://localhost:8765
uv run python -m interview_helper.main --input-file <wav>  # отладочный прогон на файле
uv run python -m interview_helper.main --list-devices      # аудиоустройства (WASAPI)
uv run python -m interview_helper.bench <wav>      # бенчмарк латентности
```

Живой запуск (`main` без `--input-file`) включает захват микрофона — его запускает пользователь, не Claude (страхует хук guard_run).

Код лежит в `src/interview_helper/`: `capture.py` (аудио, выбор устройств — только WASAPI host API) → `transcribe.py` (whisper) → `answer.py` (детекция вопроса + `claude -p`) → `pipeline.py` (общий цикл с колбэком событий). Поверх пайплайна два фронтенда: `main.py` (терминал, rich) и `web.py` + `static/index.html` (FastAPI + SSE, порт 8765). Оценка ресурсов whisper — в `docs/resources.md`.

## Стек

- **Транскрипция:** faster-whisper (локально, стриминг по чанкам).
- **Ответы:** Claude Haiku через `claude -p` (headless-режим Claude Code CLI). Пользователь использует инструмент сам для себя — это не нарушает ToS.
- **Платформа:** Windows 11. Захват системного звука — через WASAPI loopback (например, `soundcard` или `pyaudiowpatch`), микрофон — обычный устройство ввода.

## Дорожная карта

- **MVP:** рабочий инструмент без предварительного контекста — захват звука, транскрипция, ответы в реальном времени.
- **Фаза 1:** контекст по вакансии перед сессией (описание вакансии, резюме); после встречи — саммари, список вопросов и ответов, оценка выступления.
- **Фаза 2:** контекст по пользователю, накопление знаний от сессии к сессии (профиль, слабые места, история ответов).

## Архитектура (план)

Конвейер в реальном времени:

1. **Захват аудио** — два параллельных потока: loopback (звук встречи) и микрофон. Важно помечать источник — вопросы интервьюера приходят из loopback.
2. **Транскрипция** — faster-whisper обрабатывает чанки с VAD; латентность важнее полноты, поэтому маленькая модель (`small`/`base`) с `compute_type="int8"` предпочтительнее точной, но медленной.
3. **Детекция вопроса + ответ** — фрагменты транскрипта отправляются в `claude -p --model haiku` с накопленным контекстом разговора. Задержка на этом шаге критична: держать промпт коротким, использовать `--output-format stream-json` для стриминга ответа в UI.
4. **UI** — вывод ответа пользователю по мере генерации (терминал или оверлей).

Ключевые ограничения, влияющие на все решения: сквозная латентность (звук → ответ на экране) и устойчивость к перекрывающейся речи из двух источников.
