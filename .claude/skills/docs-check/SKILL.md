# Docs check

1. Посмотри, что менялось: `git diff --stat HEAD` + `git log --oneline -5 -- src/`.
2. Сверь изменённый код с документацией:
   - **README.md** — команды запуска/разработки, флаги CLI (`main.py --help`), требования.
   - **CLAUDE.md** — архитектура пайплайна, стек, дорожная карта (фаза завершена → отметить).
   - **docs/resources.md** — дефолтная модель whisper, compute_type, зависимости, влияющие на ресурсы.
3. Найденные расхождения исправь сразу (мелкие правки) — это не требует подтверждения. Крупные перестройки доков — сначала опиши пользователю.
4. Если расхождений нет — коротко скажи «доки актуальны» и ничего не меняй.
