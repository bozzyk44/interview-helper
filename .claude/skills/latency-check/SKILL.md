# Latency check

Латентность «звук → ответ на экране» — главная метрика проекта. Этот скилл замеряет её по этапам.

## Шаги

1. Определи тестовый файл: аргумент скилла, иначе `tests/fixtures/question.wav`. Если файла нет — скажи пользователю записать 5–10-секундный WAV с вопросом на английском/русском и положить его туда.
2. Запусти `uv run python -m interview_helper.bench <wav>` (если модуля bench ещё нет — создай его: он должен прогнать WAV через `transcribe.py`, затем текст через `answer.py`, замеряя `time.perf_counter()` на границах этапов).
3. Выведи таблицу: загрузка модели (отдельно, one-time), транскрипция, время до первого токена Claude, полный ответ, сквозная задержка без загрузки модели.
4. Сравни с целевым бюджетом: транскрипция чанка < 1 с, первый токен < 2 с, сквозная < 3 с. Отметь этапы, вышедшие за бюджет, и предложи, что крутить (размер модели whisper, compute_type, длина промпта).
